# v1.2 Roadmap — Dark mode

> Status: planning. Updated: 2026-06-11.
> Predecessors: [v1.0](./v1.0.md) (shipped) · [v1.1](./v1.1.md) (shipped).

## Goal

Dark mode — deferred from both v1.0 and v1.1 as "its own design cycle" — plus the two
accessibility debts carried on the books. Everything stays **additive** under
[ADR-0010](../adr/0010-semver-policy.md): npm packages 1.1.0 → **1.2.0** (MINOR),
`willink_theme` → **1.5.0** (independent, [ADR-0011](../adr/0011-flutter-independent-versioning.md)).

## Starting position (surveyed 2026-06-11)

The preset already ships a **partial** dark scaffold: `:root[data-theme="dark"]` +
`prefers-color-scheme` auto (opt-out via `data-theme="light"`), flipping 4 roles
(`bg` / `fg` / `muted` / `border`). What's missing:

- The other semantic roles don't flip (brand state tokens, feedback colors, shadows, gradient stops).
- **18 primitive-utility leaks across 11 React components** (`bg-neutral-100/200`, `hover:bg-neutral-50/100`, Tooltip's `bg-neutral-900 text-neutral-50`) that freeze light-mode grays under a dark root. The 3 `bg-black/50` overlays are intentional and stay.
- No `tokens.dark.css` export in `@willink-labs/css-tokens`.
- `WillinkTheme` is light-only (`Brightness.light` hardcoded).
- Storybook has no theme toggle.

## In scope

| Area | What ships |
|---|---|
| ADR-0013 | Dark mode strategy: formalize the shipped `data-theme` mechanism; semantic roles flip, primitives and the numeric brand scale stay invariant |
| Tokens | Dark values for every flipping semantic role via DTCG `$extensions` + new surface roles (`surface-subtle` / `surface-muted` / `track` / `surface-inverted(-fg)`) needed to absorb the primitive leaks |
| Preset | Extended dark blocks (all roles + shadows + gradient stops); new semantic utilities + safelist entries |
| css-tokens | `tokens.dark.css` export for non-Tailwind consumers |
| React | Migrate the 15 non-overlay primitive leaks to the new semantic utilities (zero `dark:` variants needed — variables do the flipping) |
| Contrast audit | `scripts/check-contrast.mjs` — computed WCAG ratios for key fg/bg pairs in both modes, wired into CI; a11y matrix gains a dark column note |
| Storybook | Theme toolbar toggle (`data-theme` decorator) so every story is reviewable in both modes |
| Flutter | `WillinkTheme.willinkDark()` (`Brightness.dark` ColorScheme, mirrors the semantic flips) → `willink_theme` 1.5.0 |
| A11y debts | Skeleton `animate-pulse` bound to `prefers-reduced-motion` (flagged since 0.13.0); white-on-danger contrast verified by the audit script (add `danger-fg` only if it fails) |

## Out of scope (v1.3 candidates)

| Item | Note |
|---|---|
| WordPress package | Next consumer group; needs a real consuming theme to pilot against. |
| `FormField` compound export | API design cycle; deferred since the 9.1 audit. |
| Flutter `Checkbox` / `Switch` / `RadioGroup` wrappers | Still covered by Material 3 natives. |
| `Popover` / `Combobox` / `Breadcrumb` / `Stepper` | Discussion-gated, unchanged. |

## Sequencing

```
roadmap ─→ ADR-0013 + tokens/preset/css-tokens (core) ─→ ┬─ React leak migration + Skeleton fix
                                                          ├─ Storybook theme toggle      (parallel)
                                                          └─ Flutter willinkDark()       (parallel)
                                                                       └─→ v1.2.0 + flutter-v1.5.0 cut
```

Core must land first — the React migration consumes the new semantic utilities.
Verification per change type follows [ADR-0012](../adr/0012-release-verification-policy.md).
