## Outcome (recorded at release)

1. **The dark contract now holds for *gradient* text, not just flat roles — and the audit can finally see it.** `text-gradient-primary` is a `bg-clip-text text-transparent` heading painted by a `brand → brand-glow` gradient (brand-600 → brand-500, two *fixed* brand primitives). By [ADR-0013](../adr/0013-dark-mode.md) the numeric brand ramp does not flip, so on the dark `bg` (`neutral-950`) the clipped heading rendered its **worst endpoint at brand-600 = 3.54:1 — below AA**, washing out. This is the same root cause as v1.7's #58 (a fixed brand foreground on a flipping surface), but hidden inside a gradient utility. The fix makes the endpoints **dark-aware** via two preset-internal vars (`--color-gradient-primary-from/-to`): **light is byte-identical** (`brand` → `brand-glow`), **dark brightens to `brand-300 → brand-400`** (worst endpoint **7.41:1**, AAA-adjacent) — the same "lighter brand step in dark" move `brand-soft-fg` already makes. The brand stays violet; only the dark legibility changes. Details + before/after: [ADR-0018](../adr/0018-gradient-accent-text-dark.md).
2. **The systemic win is closing the audit blind spot the CEO caught twice.** `scripts/check-contrast.mjs` only knew flat `--color-*` roles, so it could not see a clipped gradient heading's endpoints — exactly the defect class the i-willink.com production sweeps logged as "bg-clip-text gradient headings (audit blind spot)". The script now carries a **`TEXT_GRADIENTS` registry**: every text-clipped gradient `@utility` is declared with its endpoints per mode, and the gate asserts the **worst endpoint** clears the floor against `bg` — **required ≥ 4.5 in dark** (FAIL → CI red), **report-only baseline in light** (the legacy fixed pair ships byte-identical at 4.23:1, a documented large-text baseline like the feedback colors on white). Reverting the dark endpoints to the old fixed brand pair turns the gate **red** (3.54:1 ✗ FAIL, exit 1) — a recorded demonstration that the class can no longer regress silently. The existing per-pair checks are untouched; this is purely additive coverage.
3. **The accent-on-gradient class — the second flavor the reviews flagged — is closed as a written consumer contract, because the DS can't enumerate it.** Accent text whose hue/lightness is too close to a gradient *background* (the i-willink.com hero) is a *consumer composition*, not a DS token. The DS places no accent text on its own gradients (only white on the vivid bg-gradients, intentional). So a new [a11y gradient/accent contract](../a11y/gradient-and-accent.md) states the rule — dark-aware endpoints + hue/lightness separation; the audit enforces it for DS utilities; consumers ensure it for their own on-gradient text — naming the exact i-willink.com hero failure so it can't recur quietly across consumers. (The i-willink.com hero itself is a crew consumer-side fix; this cut does not touch i-willink.com.)
4. **npm cut is lockstep 1.7.0** with real content only in `tailwind-preset` (two preset-internal gradient vars + dark flips + the re-pointed utility) and the repo-level audit/test. `tokens` / `css-tokens` / `react` carry lockstep markers — the gradient vars are **preset-internal** (like `--color-gradient-subtle-end` since 1.2): not in `semantic.json`, not in the css-tokens output, not referenced by any component. No `flutter-v*` tag (zero Flutter changes). Verification per [ADR-0012](../adr/0012-release-verification-policy.md) Layer 1 (the gradient-aware contrast gate, both modes) + Layer 2 (this cut).

## Goal

Close the `bg-clip-text` gradient-heading dark-legibility defect in the DS's own `text-gradient-primary` utility, and — the systemic improvement — extend the contrast audit so it can *see* gradient text and freeze the class in CI, so this cannot recur across consumers. Decisions: [ADR-0018](../adr/0018-gradient-accent-text-dark.md).

Everything is additive under [ADR-0010](../adr/0010-semver-policy.md): npm packages 1.6.0 → **1.7.0** (MINOR — new `@theme` vars; light byte-identical, dark a contrast improvement, compile-safe). No Flutter changes this cycle.
