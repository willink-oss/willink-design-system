# v1.3 Roadmap — WordPress consumption

> Status: **shipped** — npm packages 1.3.0 (tag `v1.3.0`), released 2026-06-11. No Flutter release this cycle (`willink_theme` stays at 1.5.0 — independent cadence per [ADR-0011](../adr/0011-flutter-independent-versioning.md)). Updated: 2026-06-11.
> Predecessors: [v1.0](./v1.0.md) · [v1.1](./v1.1.md) · [v1.2](./v1.2.md) (all shipped).

## Outcome (recorded at release)

1. **The "WordPress package" roadmap item closes without a package.** Two pilots (CEO-approved scope, both merged) proved `@willink-labs/css-tokens` is the WordPress integration: `wp-modern-starter-kit` (canonical wiring pattern) and `esperanza-wp-theme` (production — 10/10 variable parity vs its hand-maintained SCSS copy, byte-identical dist except inert variable additions, 4/4 visual regression). Decision and evidence: [ADR-0014](../adr/0014-wordpress-consumption.md).
2. **Both pilot findings fixed in `css-tokens` 1.3.0**: root-level proxy CSS files (the postcss-import / `exports`-map blocker — documented specifiers are now physical files) and the color-free `tokens.primitives.css` (radius + duration + easing only; esperanza's radius/motion-only contract no longer ingests 50 inert color/shadow vars).
3. **npm cut is lockstep 1.3.0** with real content only in `css-tokens`; `tokens` / `tailwind-preset` / `react` carry lockstep markers (no source change). Consumer-side verification (esperanza parity / dist / visual checks) ran in that repo under the CEO flow — this repo verified the package surface per [ADR-0012](../adr/0012-release-verification-policy.md).
