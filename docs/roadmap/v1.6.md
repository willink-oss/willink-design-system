# v1.6 Roadmap — `fg-*` text-emphasis roles upstreaming

> Status: **shipped** — npm packages 1.5.0 (tag `v1.5.0`), released 2026-06-13. No Flutter release this cycle (`willink_theme` stays at 1.5.0 — independent cadence per [ADR-0011](../adr/0011-flutter-independent-versioning.md)). Updated: 2026-06-13.
> Predecessors: [v1.0](./v1.0.md) · [v1.1](./v1.1.md) · [v1.2](./v1.2.md) · [v1.3](./v1.3.md) · [v1.4](./v1.4.md) · [v1.5](./v1.5.md) (all shipped).

## Outcome (recorded at release)

1. **A consumer-grown scale becomes an official DS contract.** The five `fg-*` roles i-willink.com invented during its dark rollout (v1.5 Outcome 3) ship upstream verbatim — same names, same neutral-step mapping, same dark flips ([ADR-0016](../adr/0016-text-emphasis-roles.md)). The pattern held: the DS shipped a minimal two-tier `fg`/`muted` surface, a real consumer needed the in-between ladder, built it locally, the DS observed and promoted it. Because the DS neutral scale **is** the slate scale i-willink.com used, the upstream roles are byte-identical in light to the local definitions — the later consumer migration is a pure find-and-replace with no visual diff.
2. **The roles ride the existing machinery with zero new mechanism.** Each is a neutral alias + `willink.dark` extension, so the [ADR-0013](../adr/0013-dark-mode.md) dark-flip (dual preset blocks + `tokens.dark.css`), the consumer override surface, and the contrast gate all absorbed them without special-casing. The only genuinely new design work was the **per-role contrast policy**: `fg-strong`/`fg-emphasis` held to AAA (≥ 7), `fg-secondary` to AA (≥ 4.5), and `fg-subtle`/`fg-faint` documented as report-only baselines below the body-text floor — a captions/disabled tier, not content text. The audit now answers "is this role body-text-safe?" per rung instead of with one threshold.
3. **npm cut is lockstep 1.5.0** with real content in `tokens` (5 roles), `tailwind-preset` (light decls + both dark blocks + `text-fg-*` safelist) and `css-tokens` (regen: 5 new vars in light + dark output); `react` carries a lockstep marker (no component change — consumers opt in via `text-fg-*` utilities). No `flutter-v*` tag (zero Flutter changes). Verification per [ADR-0012](../adr/0012-release-verification-policy.md) Layer 1 (token-row: tokens tests + contrast gate both modes) and Layer 2 (this cut).

## Goal

Upstream the `fg-*` text-emphasis ladder flagged in [v1.5](./v1.5.md) Outcome 3 into the DS as official semantic roles, so every consumer with more than a two-tier type hierarchy gets a foreground emphasis ladder that **flips in dark mode** — removing the only remaining reason a DS consumer reaches into mode-invariant `text-neutral-*` for body copy. Naming + contrast rationale: [ADR-0016](../adr/0016-text-emphasis-roles.md).

Everything is additive under [ADR-0010](../adr/0010-semver-policy.md): npm packages 1.4.1 → **1.5.0** (MINOR). No Flutter changes this cycle.

## Scope

| Area | What ships |
|---|---|
| `tokens` | 5 new semantic roles between `fg` and `muted`: `fg-strong` (n800/dark n100) · `fg-emphasis` (n700/n200) · `fg-secondary` (n600/n300) · `fg-subtle` (n400/n500) · `fg-faint` (n300/n600). Each a neutral alias + `willink.dark` extension. `fg` / `muted` unchanged (the ladder's anchors) |
| Preset | 5 light `--color-fg-*` decls + the same 5 in **both** dark blocks (textually in sync per ADR-0013); `text-fg-*` safelist entries |
| css-tokens | regen — 5 vars in `tokens.css` / `tokens.semantic.css`, flipped in `tokens.dark.css` (+ root proxies) |
| Verification | `scripts/check-contrast.mjs` + 5 role/bg pairs, both modes (strong/emphasis ≥ 7, secondary ≥ 4.5 required; subtle/faint report-only baselines); token schema + css-tokens generated tests assert light + dark |
| Docs | [ADR-0016](../adr/0016-text-emphasis-roles.md); README roadmap line + css-tokens / preset role notes; [a11y matrix](../a11y/matrix.md) adopter note on the ladder's contrast floor |

No React component changes: consumers opt in via `text-fg-*` utilities; the frozen component surface does not move.
