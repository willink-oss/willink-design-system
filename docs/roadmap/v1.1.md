## Goal

v1.0.0 shipped the API freeze for the five npm-side packages and `willink_theme` (see [v1.0 roadmap — Outcome](./v1.0.md#outcome-recorded-after-release)). Two planned phases were cut from that release to keep the freeze date: Storybook documentation and the Flutter component expansion. **v1.1** delivers those two carried-over items.

Post-1.0, every change in this cycle must be **additive** under [ADR-0010](../adr/0010-semver-policy.md): the npm packages move 1.0.0 → 1.1.0 (MINOR), and `willink_theme` versions independently per [ADR-0011](../adr/0011-flutter-independent-versioning.md).

## Guiding principles

- **Strict SemVer now applies.** No breaking changes ride along in a minor. Anything that would require a MAJOR bump is out of scope for v1.1 by definition.
- **No tooling without same-PR content.** The Phase 0 rule still applies — the Storybook bootstrap PR must include stories, not an empty scaffold.
- **Independent Flutter cadence** ([ADR-0011](../adr/0011-flutter-independent-versioning.md)). Flutter-only PRs bump `willink_theme` alone (1.1.0, 1.2.0, … as components land); they do not touch the npm packages.

## Release shape

- npm packages: a single lockstep 1.0.0 → **1.1.0** MINOR cut once the Storybook stories are complete. (Storybook itself is an `apps/` workspace, not a published package — if no package surface changes by then, the npm bump may be skipped and v1.1 becomes a docs/tooling milestone; decide at cut time.)
- `willink_theme`: independent MINOR bumps as each of the four components lands (1.1.0 → 1.4.0 expected), each with its own `flutter-v1.x.0` tag and pub.dev publish.

## Sequencing

```
Storybook bootstrap (≥3 stories) ─→ story batches of 5 ─→ a11y addon cross-link ─┐
                                                                                  ├─→ v1.1 cut
TabBar ─→ BottomSheet ─→ SnackBar ─→ ProgressIndicator   (flutter, independent) ─┘
```

The two tracks are fully parallel-safe; neither blocks the other.
