> Status: **shipped** — npm packages 1.6.0 (tag `v1.6.0`), released 2026-06-13. No Flutter release this cycle (`willink_theme` stays at 1.5.0 — independent cadence per [ADR-0011](../adr/0011-flutter-independent-versioning.md)). Updated: 2026-06-13.
> Predecessors: [v1.0](./v1.0.md) · [v1.1](./v1.1.md) · [v1.2](./v1.2.md) · [v1.3](./v1.3.md) · [v1.4](./v1.4.md) · [v1.5](./v1.5.md) · [v1.6](./v1.6.md) (all shipped).

## Outcome (recorded at release)

1. **The dark contract now holds for brand *text*, not just surfaces.** v1.2 made every *surface* flip; this cut closes the matching gap for brand *foreground text*. The `Button variant="link"` resting color was `text-brand` — the fixed `brand-600` primitive, which by [ADR-0013](../adr/0013-dark-mode.md) does not flip — so on the dark page background it rendered **3.54:1, below AA** (#58, caught by the i-willink.com production sweep on `/design-system`). Re-pointing it to the flipping `text-brand-soft-fg` (light `brand-700` 7.10:1 / dark `brand-300` 10.93:1) fixes both modes to AAA. The principle that decided it is ADR-0013's: a role on a flipping surface must itself flip — a fixed primitive there is always a latent dark bug.
2. **The bug was treated as a class and cleared DS-wide.** Auditing every fixed brand foreground in the React sources found one more instance — `AccordionTrigger`'s `hover:text-brand` (3.54:1 on hover in dark) → `hover:text-brand-hover` (4.76:1, AA). `RadioGroupItem`'s `text-brand` was audited and **kept**: it colors an 8 px indicator dot (a graphical object under WCAG 1.4.11's 3:1 rule, which it clears at 3.54:1), not text. Two new **required** contrast-gate pairs (`brand-soft-fg / bg`, `brand-hover / bg`, both modes) freeze the class so it can't silently regress.
3. **`info-fg` was evaluated and deferred — minimalism with evidence.** i-willink.com's local `--color-info-fg` (blue, dual-block) was assessed for upstreaming (#59). Verdict **NO this cycle**: it has a *single* usage site (one decorative heading), there is no `info` hue or surface anywhere in the DS for it to belong to (it would ship as an orphan blue text role, not a feedback family), and the promote-on-reproduction bar that justified the [ADR-0016](../adr/0016-text-emphasis-roles.md) `fg-*` ladder (structural + pervasive + reproduced) is unmet. #59 is **left open** with the reasoning as a comment — a defer, not a reject. The contrast between #58 (a real bug, fixed with *zero* new tokens) and #59 (a role that hasn't earned its keep) is the cycle's theme: fix what's broken, don't grow the surface without evidence.
4. **npm cut is lockstep 1.6.0** with real content only in `react` (the two component fixes, both built on roles that already existed). `tokens` / `tailwind-preset` / `css-tokens` carry lockstep markers — the fix added no token, no `@theme` variable, no safelist entry, no css-tokens var (all already present from v1.2). No `flutter-v*` tag (zero Flutter changes). Verification per [ADR-0012](../adr/0012-release-verification-policy.md) Layer 1 (modified-component: affected unit tests, story note, a11y-matrix rows) + the contrast gate, and Layer 2 (this cut).

## Goal

Close [#58](https://github.com/willink-oss/willink-design-system/issues/58) (Button `link` AA failure in dark) and clear the whole "fixed brand foreground on a flipping surface" bug class, and make a recorded call on [#59](https://github.com/willink-oss/willink-design-system/issues/59) (`info-fg` upstreaming). Decisions: [ADR-0017](../adr/0017-dark-link-contrast-and-info-fg.md).

Everything is additive/fix-grade under [ADR-0010](../adr/0010-semver-policy.md): npm packages 1.5.0 → **1.6.0** (MINOR — the link variant's default visual changes, compile-safe). No Flutter changes this cycle.

| Area | What ships |
