# v1.4 Roadmap — FormField compound

> Status: **in progress** — target: npm lockstep 1.3.0 → 1.4.0 (tag `v1.4.0`). Updated: 2026-06-11.
> Predecessors: [v1.0](./v1.0.md) · [v1.1](./v1.1.md) · [v1.2](./v1.2.md) · [v1.3](./v1.3.md) (all shipped).

## Goal

Close the longest-deferred React roadmap item: the `FormField` compound export, deferred at the v1.0 Phase 9.1 API audit and carried through v1.1–v1.3 as "API design cycle". The deferral discipline pays off as a deliberately **minimal** surface — a11y wiring only, no form-state opinion. API decisions: [ADR-0015](../adr/0015-formfield-api.md).

Everything is additive under [ADR-0010](../adr/0010-semver-policy.md): npm packages 1.3.0 → **1.4.0** (MINOR). No Flutter changes this cycle.

## Scope

| Area | What ships |
|---|---|
| React | `FormField` / `FormFieldLabel` / `FormFieldControl` (Radix Slot) / `FormFieldDescription` / `FormFieldError` + exported `*Props` types — generated ids, label `htmlFor`, merged `aria-describedby` (rendered nodes only), auto `aria-invalid`, `role="alert"` error. 25th component; zero new runtime deps |
| Preset | `text-danger` safelist entry (the one new class string crossing the dist boundary) — real `tailwind-preset` CHANGELOG entry, not a lockstep marker |
| Verification | Per [ADR-0012](../adr/0012-release-verification-policy.md) component-new row: 12 unit tests incl. jest-axe, Storybook story (a11y addon surface), playground demo, [a11y matrix](../a11y/matrix.md) row (React table → 25) |
| Docs | [ADR-0015](../adr/0015-formfield-api.md); README component tables; `packages/react/README.md` retires the "write your own FormField" pattern in favor of the shipped compound |

## Out of scope (v1.5 candidates)

| Item | Note |
|---|---|
| Dark-mode consumer rollout | Mechanism shipped in v1.2 ([ADR-0013](../adr/0013-dark-mode.md)); driving adoption across the consumer fleet is its own cycle. |
| Flutter `Checkbox` / `Switch` / `RadioGroup` wrappers | Still covered by Material 3 natives. |
| `Popover` / `Combobox` / `Breadcrumb` / `Stepper` | Discussion-gated, unchanged. |

## Release shape

Single lockstep MINOR cut 1.3.0 → **1.4.0** (npm group only, tag `v1.4.0`). Real content in `react` (the compound) and `tailwind-preset` (safelist entry); `tokens` / `css-tokens` carry lockstep markers. No `flutter-v*` tag: zero Flutter changes, and [ADR-0011](../adr/0011-flutter-independent-versioning.md) rejects content-free catch-up releases on the Flutter line. Verification per [ADR-0012](../adr/0012-release-verification-policy.md) Layer 2.
