# v1.4 Roadmap — FormField compound

> Status: **shipped** — npm packages 1.4.0 (tag `v1.4.0`), released 2026-06-11. No Flutter release this cycle (`willink_theme` stays at 1.5.0 — independent cadence per [ADR-0011](../adr/0011-flutter-independent-versioning.md)). Updated: 2026-06-11.
> Predecessors: [v1.0](./v1.0.md) · [v1.1](./v1.1.md) · [v1.2](./v1.2.md) · [v1.3](./v1.3.md) (all shipped).

## Outcome (recorded at release)

1. **The longest-deferred React item closes as designed — minimal.** The compound shipped as exactly five subcomponents + five `*Props` types ([ADR-0015](../adr/0015-formfield-api.md)), zero new runtime deps (Radix `Slot` reuse), no CVA, no form-state coupling — the two deferrals bought a surface that wires a11y and nothing else. The three places that documented the hand-wiring pattern (package README "DIY FormField", Input `Invalid` story, playground) now point at / demonstrate the shipped compound instead.
2. **One design judgment worth recording**: description/error presence is detected by render-time inspection of `FormField`'s *direct children* — chosen over effect-based registration (SSR first paint would lack the aria attributes) and over shadcn-style unconditional id emission (dangling `aria-describedby` fails axe). The direct-children constraint is the documented sharp edge; escalation path if real consumers need nesting is an explicit registration API, per the ADR.
3. **npm cut is lockstep 1.4.0** with real content in `react` (the compound, 12 new unit tests → 171 total) and `tailwind-preset` (`text-danger` safelist entry); `tokens` / `css-tokens` carry lockstep markers. No `flutter-v*` tag (zero Flutter changes). Verification ran per [ADR-0012](../adr/0012-release-verification-policy.md) Layer 0/1 (component-new row, PR #51) and Layer 2 (this cut).

## Goal

Close the longest-deferred React roadmap item: the `FormField` compound export, deferred at the v1.0 Phase 9.1 API audit and carried through v1.1–v1.3 as "API design cycle". The deferral discipline pays off as a deliberately **minimal** surface — a11y wiring only, no form-state opinion. API decisions: [ADR-0015](../adr/0015-formfield-api.md).

Everything is additive under [ADR-0010](../adr/0010-semver-policy.md): npm packages 1.3.0 → **1.4.0** (MINOR). No Flutter changes this cycle.
